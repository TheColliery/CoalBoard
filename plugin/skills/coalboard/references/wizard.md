# CoalBoard — the manual `/coalboard` setup wizard

**Route by SIGNAL, not by a fixed default:** a bare `/coalboard` with a lay phrasing ("check my project", no config words) → LAYMAN. A `/coalboard` with a TECHNICAL target (a repo/subproject/path, a rigor/depth/lens word, stated prefs) → PROGRAMMER — a technical user OFFERED the picks (order→bill→pay), never silently auto-picked-then-billed. When unsure, lean PROGRAMMER for a technical target (offering picks is cheap; a surprise auto-config the user must undo is not).

## LAYMAN-DEFAULT — the AI handles everything (the default path)
1. **TARGET:** a path/cwd given → use it; else ONE plain question ("what should I check?" — cwd ✓ · a folder · the clean mirror).
2. **Scan silently** (enumerate-only + the exclude-floor below) → SAFE defaults: work-type from the scan · depth L2 · rigor standard · dispatch all-at-once.
3. **ONE plain bill = the only question — the DETAIL renders as chat text BEFORE the box, the box holds only the decision line + options** (anti-rubber-stamp — never cram the breakdown INTO the question). Chat text: `[target] · [N] files · 3 independent reviewers + a judge · ~[Y] tokens`. Box: a 1-line summary + `[Go ✓ · cheaper · more thorough · cancel]`. **MARK the auto-picks as CHANGEABLE** — the box names `cheaper`/`more thorough` as the change levers (they map to depth/rigor silently; NO opaque jargon), so the user knows the config was chosen FOR them and is theirs to change. Picking `cheaper`/`more thorough` = a CHANGE → RECOMPUTE → re-render (chat detail + a fresh box); only `Go` on the SHOWN bill convenes.
4. On Go → convene. (Safety = SKILL.md Step 4 staging + human-apply gate.)
- **Layman honesty (NEVER false confidence):** the result says what it checked + RAN + found AND what it could NOT verify, plus "for anything irreversible (money / security / data-loss) get a human / second opinion" — NEVER "trust me".

## PROGRAMMER — full control (opt-in: "advanced" / stating prefs / config), the restaurant flow
**order → bill → pay = 3 boxes** — the bill is computed AFTER the picks, so the post-pick confirm is its OWN box, NEVER folded into the settings box.
- **Call 1 — TARGET (asked FIRST, before any scan):** cwd ✓ · clean mirror (`Colliery/`) · subproject · path · diff · configured. Then read VERSION from the target's OWN files; note SOURCE vs install-clone (sets report location — `audit.md`).
- **Scan (silent · enumerate-only · ONE line):** classify by extension/path, NO content read. EXCLUDE = `excludePaths` (config) ∪ the dev-contamination floor (`CLAUDE.md`/`MEMORY.md`/`AGENTS.md`/`.claude/`/`.agents/`); `.github`/`workflows/` stay IN-scope (`audit.md`). FRESH (no prior-`audit-*.md` read). Surface: `Scanned: N files · class · source|clone · report→path · prior-audit?`.
- **Call 2 — the 3 settings (batched):** WORK-TYPE (scan-derived · ASK — the only scope-narrowing) · DEPTH (L1 · L2 ✓ · L3 ~3-4×) · RIGOR (relaxed · standard ✓ · high · nasa; per-instance, never written to config).
- **Bill → Call 3 — confirm:** compute the cost FROM the picks. **Render the full bill as chat text BEFORE the box** (target · N files · resolved config · model-per-lens · ~cost · any warnings) — the box is NOT the place for the breakdown (anti-rubber-stamp: a wall of detail crammed into a question defeats the read). **Call 3's box carries only a 1-2 line decision summary** (`~cost · headline config, e.g. nasa/L3`) + ONE question: confirm ✓ · change · cancel. **On `change`: loop back to Call 2 (re-pick) → RECOMPUTE → re-render (chat bill + a FRESH box)** — a change can multiply cost (e.g. → nasa), so NEVER spawn on a stale bill and NEVER fold the recompute into the change step. The loop is `order → bill → change → bill → pay`. Spawn the first lens ONLY on a confirm of the CURRENT bill. DISPATCH defaults all-at-once (config-overridable; ask only if raised). Boarding-scope per `audit.md`.

## Convene
Substitute the resolved values into `references/lens-prompts.md` (it owns the fill rules); model = the per-seat `rigorLensTiers` ladder (SKILL.md Step 1). If the resolved models seat **fable** (high/nasa), SKILL.md Step 1 fires the fable consent-gate BEFORE spawning those seats (`references/platform-cc.md` owns the cost/rate note). Convene (SKILL.md Steps 1–4).
